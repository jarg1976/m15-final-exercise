TechPath Mentoria

Enunciado do Projeto:

A TechPath Mentoria é uma empresa inovadora que oferece um serviço de acompanhamento personalizado a estudantes e profissionais da área de tecnologia que querem aprimorar as suas competências técnicas e comportamentais. O serviço estabelece uma ligação entre alunos e mentores experientes, que avaliam o progresso de cada aluno em sessões periódicas, oferecendo feedback detalhado e dicas de melhoria.

Para suportar este serviço, a TechPath Mentoria precisa de uma base de dados que organize todas as informações sobre alunos, mentores, sessões de mentoria e avaliações. Esta base de dados ajudará a empresa a monitorizar o progresso dos alunos, identificar os que se destacam e fornecer relatórios mensais de desempenho.

Requisitos de negócio para a Base de Dados

1. Registo de Alunos: A empresa quer manter registos de todos os alunos inscritos no programa de mentoria. Para cada aluno, é importante armazenar informações de contacto, dados pessoais e a data em que o aluno se inscreveu no programa.

2. Informações sobre Mentores: Cada mentor possui uma área de especialização, como “Programação”, “Data Science” ou “Comunicação Interpessoal”, e atua em várias sessões de mentoria com diferentes alunos. Para cada mentor, é necessário manter dados básicos de contacto e a sua área de atuação.

3. Sessões de Mentoria: As sessões de mentoria são realizadas regularmente entre mentores e alunos. Para cada sessão, a empresa quer registar a data em que ocorreu, o tema abordado e uma avaliação do desempenho do aluno, atribuída pelo mentor numa escala de 1 a 10. Esse acompanhamento é essencial para entender a evolução do aluno ao longo do tempo.

4. Histórico de Avaliações: A Mentoria Conectada deseja também manter um histórico mensal das avaliações de cada aluno, permitindo calcular uma média do seu desempenho ao longo dos meses. Este histórico servirá para filtrar os alunos com melhor desempenho, definindo um padrão de excelência para o programa.

Objetivos para o Projeto

. Identificar as entidades e relações necessárias para a base de dados, construindo um Modelo ER que respeite os requisitos de negócio.

. Criar um script SQL para construir a base de dados, incluindo as tabelas e colunas definidas com base no modelo ER e selecionando os tipos de dados mais adequados.

. Incluir chaves primárias, estrangeiras e índices que otimizem a estrutura e garantam a integridade referencial da base de dados.

. Inserir registos de exemplo para cada tabela criada na base de dados. O mínimo necessário inclui: 3 mentores, cada um com uma área de especialização distinta; 3 alunos, com dados pessoais completos e datas de inscrição; pelo menos 6 sessões de mentoria, que devem associar mentores a alunos e incluir as datas, temas abordados e uma avaliação do desempenho de cada aluno. Além disso, deve haver registos suficientes no histórico de avaliações para calcular uma média mensal de desempenho para cada aluno, possibilitando a análise da sua evolução.

Este projeto permitirá à empresa identificar os alunos com maior potencial e acompanhar o desenvolvimento contínuo de cada um, com base nos dados registados sobre a participação e avaliações ao longo dos programas de mentoria.
